# SenseAR Effects SDK 合规使用指南

更新日期：2026 年 4 月 28 日

## 一、前言

为帮助开发者在遵守个人信息保护相关法律法规、监管要求及平台规范的前提下合规接入、使用 **SenseAR Effects SDK**（以下简称“本 SDK”），北京大甜绵白糖科技有限公司制定本合规使用指南。

开发者在接入、集成、调用、分发本 SDK 前，应充分阅读本指南、SDK 许可协议及相关技术文档，并根据自身 App 的实际业务场景、产品形态、权限申请、个人信息处理情况，完善 App 内的隐私政策、权限弹窗、用户授权流程及其他合规措施。

本指南仅用于说明本 SDK 的基本功能、权限使用、个人信息处理及接入合规建议，不替代开发者自身作为 App 开发运营者应承担的合规义务。

## 三、SDK 基本业务功能说明

本 SDK 为开发者提供美颜、美型、微整形、美妆、滤镜、贴纸、美体等特效渲染能力。开发者可将本 SDK 集成至授权 App 或项目中，用于对开发者传入的图像、视频帧或相关渲染输入进行本地特效处理。

| 业务功能 | 功能介绍 | 是否基础功能 |
| --- | --- | --- |
| 美颜美型 | 对图像或视频画面进行美白、磨皮、肤色优化、五官调整等效果处理 | 是 |
| 微整形 | 对人脸关键区域进行视觉效果调整 | 是 |
| 美妆 | 对画面中的人脸添加口红、腮红、眼妆等视觉效果 | 是 |
| 滤镜 | 对图像或视频画面添加整体风格化滤镜效果 | 是 |
| 贴纸/道具 | 对画面添加贴纸、道具或互动视觉效果 | 是 |
| 美体 | 对人体相关画面进行视觉效果调整 | 是 |

开发者应确保仅在终端用户主动使用相关拍摄、编辑、预览、渲染或特效功能时调用本 SDK。若 App 自身需要使用相机、麦克风、相册、文件读写等系统能力，应由开发者根据 App 实际功能依法另行申请对应权限，并在 App 隐私政策及权限弹窗中向终端用户充分告知。

## 四、接入或升级至合规版本

开发者应使用本 SDK 的正式授权版本，并按照双方约定的授权范围进行集成和分发。建议开发者关注 SDK 提供方发布的版本更新信息，在需要获取新功能、修复缺陷、满足平台或监管要求时及时升级。

开发者不得使用来源不明、未经授权、被篡改或从其他产品中反向提取的 SDK 文件。不得将 SDK 本体、库文件、模型文件、算法文件、技术资料或示例工程发布至公开代码仓库、论坛、网盘、开源平台或其他第三方可访问渠道。

## 五、SDK 个人信息处理说明

本 SDK 的核心特效能力在设备本地运行。就本 SDK 自身而言：

1. 不会主动采集终端用户姓名、手机号、身份证号、通讯录、位置信息等个人身份信息。
2. 不会主动上传终端用户的人脸图像、视频、音频、相册文件或其他个人信息至 SDK 提供方服务器。
3. 不会主动读取设备标识符、应用安装列表、通讯录、短信、通话记录、定位、剪贴板等信息。
4. 不内置广告投放、用户画像、行为追踪、数据上报或第三方统计分析能力。
5. 仅在本地处理开发者主动传入的图像、视频帧或相关渲染输入，以实现美颜、滤镜、贴纸等视觉效果。

如开发者在自身 App 中基于本 SDK 能力实现拍摄、录制、相册导入、图片编辑、视频编辑、内容保存、分享、上传等功能，相关个人信息处理行为的处理主体、处理目的、处理方式、保存期限和用户权利响应机制，应由开发者在 App 隐私政策及相关交互界面中另行向终端用户告知并依法取得必要授权。

## 六、SDK 申请系统权限说明

本 SDK 内部仅依赖以下 HarmonyOS 权限，用于保存 License 校验相关信息。

| 操作系统 | 权限名称 | 是否必需 | 使用目的 | 使用场景/申请时机 | 是否涉及个人信息 |
| --- | --- | --- | --- | --- | --- |
| HarmonyOS | ohos.permission.STORE_PERSISTENT_DATA | 必需 | 用于持久化保存 SDK License 校验信息，保障授权状态可被正常校验 | 开发者初始化 SDK 并进行 License 校验时 | 否 |

说明：

1. `ohos.permission.STORE_PERSISTENT_DATA` 仅用于保存本 SDK 的 License 校验信息，不用于保存终端用户个人信息。
2. 本 SDK 不因实现自身特效渲染功能而主动申请相机、麦克风、相册、定位、通讯录、蓝牙、日历、剪贴板、网络访问等权限。
3. 若开发者 App 为实现拍摄、录制、相册导入、文件保存、内容上传等自身业务功能，需要申请相机、麦克风、媒体读写、网络等权限，应由开发者基于实际业务功能自行完成权限声明、申请、解释和用户授权管理。
4. 开发者应避免在用户未触发相关功能、未阅读并同意 App 隐私政策或未理解权限用途前，提前、频繁或超范围申请系统权限。

HarmonyOS 权限声明示例：

```json
{
  "requestPermissions": [
    {
      "name": "ohos.permission.STORE_PERSISTENT_DATA",
      "reason": "$string:permission_store_persistent_data_reason",
      "usedScene": {
        "abilities": [
          "EntryAbility"
        ],
        "when": "inuse"
      }
    }
  ]
}
```

权限用途说明文案示例：

```text
用于保存 SenseAR Effects SDK 的 License 校验信息，以便完成 SDK 授权状态校验。
```

开发者应根据实际工程结构、HarmonyOS 平台规范及应用市场审核要求调整上述配置示例。

## 七、可选个人信息及可选权限配置说明

本 SDK 不提供可选个人信息采集开关，不存在由 SDK 自身控制的可选个人信息采集项。

本 SDK 不提供可选系统权限配置项。除 `ohos.permission.STORE_PERSISTENT_DATA` 用于 License 校验信息存储外，本 SDK 不要求开发者为 SDK 自身功能额外声明其他系统权限。

如开发者 App 的业务功能需要使用其他权限或处理其他个人信息，应在开发者自身 App 的权限配置、隐私政策、用户授权弹窗、功能开关中进行说明和控制。开发者不得将 App 自身业务所需权限表述为本 SDK 必需权限。

## 八、扩展业务功能说明

本 SDK 当前不包含独立于上述特效渲染能力之外的扩展业务功能，不提供广告、推送、账号登录、支付、数据统计、用户画像、云端识别、云端存储或内容分发等功能。

若后续版本新增可能涉及个人信息处理或系统权限申请的扩展功能，SDK 提供方将根据实际情况更新合规说明。开发者应在升级 SDK 前重新核对版本变更内容，并同步更新 App 隐私政策、权限说明及用户授权流程。

## 九、用户同意及 SDK 初始化时机

开发者应确保在 App 首次运行或首次使用相关功能前，通过简洁、明确且易于访问的方式向终端用户展示 App 隐私政策，并说明 App 集成本 SDK 的情况。

建议开发者遵守以下初始化和调用原则：

1. 在终端用户阅读并同意 App 隐私政策后，再初始化本 SDK 或调用 License 校验能力。
2. 在终端用户主动进入拍摄、编辑、预览、渲染、特效处理等相关功能场景后，再调用本 SDK 的具体业务接口。
3. 在终端用户未同意 App 隐私政策前，不得通过 App 自身逻辑申请敏感权限，不得采集、处理、上传终端用户个人信息。
4. 如终端用户拒绝非必要权限授权，开发者不得因此关闭与该权限无关的其他功能。
5. 如终端用户撤回授权或关闭相关权限，开发者应停止调用依赖该授权的 App 功能，并为用户提供合理的功能降级或提示。

开发者将 Demo 或示例代码用于生产环境前，应至少完成以下检查：

4. 确认 App 未在用户同意隐私政策前初始化 SDK、申请敏感权限或处理个人信息。
5. 确认 App 未将 SDK 文件、模型文件、算法文件、技术资料或其他保密材料暴露给未授权第三方。

## 十二、开发者合规接入注意事项

开发者在接入和使用本 SDK 时，应重点避免以下情形：

1. 未经用户同意即初始化 SDK 或调用涉及用户个人信息处理的 App 功能。
2. 未明确告知权限用途即申请权限。
3. 在用户未触发相关功能时提前申请相机、麦克风、相册、定位等敏感权限。
4. 在用户拒绝授权后频繁弹窗、反复索权或影响无关功能正常使用。
5. 超出业务必要范围收集、使用、保存或上传个人信息。
6. 将 App 自身采集、上传、分享或保存个人信息的行为表述为 SDK 行为。
7. 将本 SDK 用于授权范围之外的产品、地区、项目或用途。
8. 对本 SDK 进行反向工程、反编译、反汇编、解密、绕过保护机制或试图获取源代码、算法、模型或底层技术。
9. 未经许可向第三方提供 SDK 本体、库文件、模型文件、技术资料或其他受保护材料。
